مقدمة عن البرمجة الكائنية (OOP)

البرمجة الكائنية (Object-Oriented Programming) هي نموذج برمجة يعتمد على مفهوم "الكائنات" (Objects) التي تحتوي على بيانات ووظائف. تعتمد OOP على أربع مبادئ أساسية:

مبادئ البرمجة الكائنية

  • التغليف (Encapsulation): إخفاء التفاصيل الداخلية للكائن وعرض الواجهة فقط
  • الوراثة (Inheritance): إمكانية إنشاء فئات جديدة بناء على فئات موجودة
  • تعدد الأشكال (Polymorphism): إمكانية تنفيذ نفس العملية بطرق مختلفة
  • التجريد (Abstraction): التركيز على الخصائص الأساسية وإخفاء التفاصيل

لماذا نستخدم OOP؟

البرمجة الكائنية تساعد في تنظيم الكود، تجعله أكثر قابلية للصيانة، تسهل إعادة استخدام الكود، وتوفر نموذجًا واضحًا لتمثيل الواقع في البرمجة.

Class و Object في Dart

الفرق بين Class و Object

Class Object
قالب أو مخطط لإنشاء الكائنات نسخة ملموسة من الـ Class
يحتوي على تعريف الخصائص والوظائف يحتوي على قيم فعلية للخصائص
يتم تعريفه مرة واحدة يمكن إنشاء العديد من الكائنات من نفس الـ Class

مثال عملي 


class Student {
  // Properties (خصائص)
  String name;
  int age;
  
  // Constructor (بناء)
  Student(this.name, this.age);
  
  // Method (وظيفة)
  void introduceYourself() {
    print('Hello, my name is $name and I am $age years old.');
  }
}

// إنشاء كائن من الـ Class
void main() {
  var student1 = Student('Ahmed', 20);
  student1.introduceYourself();
}

الـ Constructors في Dart

أنواع Constructors

  • Default Constructor: المُنشئ الافتراضي الذي يأخذ نفس اسم الـ Class
  • Parameterized Constructor: مُنشئ يأخذ معاملات لتهيئة الكائن
  • Named Constructor: مُنشئ مسمى يسمح بإنشاء كائن بطرق مختلفة
  • Factory Constructor: يُستخدم عندما نريد التحكم في إنشاء الكائنات

أمثلة عملية 


class Person {
  String name;
  int age;
  
  // Default Constructor
  Person() {
    name = 'Unknown';
    age = 0;
  }
  
  // Parameterized Constructor
  Person(this.name, this.age);
  
  // Named Constructor
  Person.fromBirthYear(String name, int birthYear) {
    this.name = name;
    this.age = DateTime().now().year - birthYear;
  }
  
  // Factory Constructor
  factory Person.fromJson(Map<String, dynamic> json) {
    return Person(json['name'], json['age']);
  }
}

نصيحة مهمة

في Dart، يمكنك استخدام صيغة مختصرة لـ Parameterized Constructor كما في المثال: Person(this.name, this.age) بدلاً من كتابة التعيين يدويًا.

الـ Inheritance والـ Super

مفهوم الوراثة (Inheritance)

الوراثة تسمح لنا بإنشاء فئة جديدة (فئة ابنة) بناء على فئة موجودة (فئة أب). الفئة الابنة ترث جميع الخصائص والوظائف من الفئة الأب ويمكنها إضافة خصائص ووظائف جديدة أو تعديل الوظائف الموروثة.

مثال عملي 


class Animal {
  String name;
  int age;
  
  Animal(this.name, this.age);
  
  void eat() {
    print('$name is eating.');
  }
}

class Dog extends Animal {
  String breed;
  
  // استخدام super لاستدعاء مُنشئ الفئة الأب
  Dog(String name, int age, this.breed) : super(name, age);
  
  void bark() {
    print('$name is barking!');
  }
  
  // Override للدالة eat من الفئة الأب
  @override
  void eat() {
    super.eat(); // استدعاء الدالة من الفئة الأب
    print('$name is eating dog food.');
  }
}

void main() {
  var dog = Dog('Buddy', 3, 'Golden Retriever');
  dog.eat(); // يستدعي الدالة المعدلة
  dog.bark(); // دالة جديدة في الفئة الابن
}

ملاحظة مهمة

في Dart، تدعم الوراثة المفردة فقط، أي أن الفئة يمكن أن ترث من فئة أب واحدة فقط. لتحقيق تعددية الوراثة، يمكن استخدام Mixins.

الـ Polymorphism والـ Override

مفهوم تعدد الأشكال (Polymorphism)

تعدد الأشكال يعني قدرة الكائنات المختلفة على الاستجابة لنفس الرسالة (استدعاء نفس الدالة) بطرق مختلفة حسب نوع الكائن. يتم تحقيق ذلك عادةً عن طريق Override للدوال الموروثة من الفئة الأب.

مثال عملي 


abstract class Shape {
  void draw(); // دالة مجردة بدون تنفيذ
}

class Circle extends Shape {
  @override
  void draw() {
    print('Drawing a circle');
  }
}

class Square extends Shape {
  @override
  void draw() {
    print('Drawing a square');
  }
}

void main() {
  List<Shape> shapes = [Circle(), Square()];
  
  for (var shape in shapes) {
    shape.draw(); // نفس الاستدعاء، نتائج مختلفة
  }
}

متى نستخدم @override؟

يجب استخدام annotation @override عند تعديل دالة موروثة من الفئة الأب. هذا يجعل الكود أكثر وضوحًا ويساعد في اكتشاف الأخطاء أثناء الترجمة إذا لم تكن الدالة موجودة في الفئة الأب.

الـ Abstract Classes والـ Interfaces

الفرق بين Abstract Class و Interface

Abstract Class Interface
يمكن أن تحتوي على دوال مجردة ودوال مكتملة تحتوي فقط على تعريفات للدوال (مثل العقد)
يمكن أن تحتوي على خصائص لا تحتوي على خصائص
يمكن أن تحتوي على Constructors لا يمكن أن تحتوي على Constructors
الفئة يمكن أن ترث من Abstract Class واحدة فقط الفئة يمكن أن تطبق عدة Interfaces

أمثلة عملية 


// Abstract Class
abstract class Vehicle {
  String name;
  
  Vehicle(this.name);
  
  // Abstract method (يجب تنفيذها في الفئات الابن)
  void start();
  
  // Concrete method (تنفيذ مكتمل)
  void stop() {
    print('$name has stopped.');
  }
}

// Interface (في Dart، كل Class هو Interface)
class Drivable {
  void accelerate() {
    // تنفيذ افتراضي
  }
}

class Car extends Vehicle implements Drivable {
  Car(String name) : super(name);
  
  @override
  void start() {
    print('$name car engine started.');
  }
  
  @override
  void accelerate() {
    print('$name is accelerating.');
  }
}

ملاحظة حول Dart

في Dart، لا يوجد مفهوم منفصل للـ Interface كما في بعض اللغات الأخرى. بدلاً من ذلك، كل Class يعتبر أيضًا Interface يمكن تطبيقه باستخدام الكلمة المفتاحية implements.

الـ Encapsulation والـ Setters/Getters

مفهوم التغليف (Encapsulation)

التغليف هو مبدأ إخفاء التفاصيل الداخلية للكائن وعرض واجهة للتعامل معه فقط. يتم تحقيق ذلك عادةً عن طريق:

  • جعل الخصائص private (باستخدام _ قبل اسم الخاصية)
  • توفير دوال getter و setter للتحكم في الوصول
  • تعريف واجهة واضحة للتفاعل مع الكائن

مثال عملي 


class BankAccount {
  String _accountNumber; // خاصية خاصة
  double _balance;
  
  BankAccount(this._accountNumber, this._balance);
  
  // Getter للرصيد
  double get balance => _balance;
  
  // Setter للرصيد (مع التحقق)
  set balance(double newBalance) {
    if (newBalance >= 0) {
      _balance = newBalance;
    } else {
      print('Balance cannot be negative');
    }
  }
  
  // طريقة للإيداع
  void deposit(double amount) {
    if (amount > 0) {
      _balance += amount;
      print('Deposited $amount. New balance: $_balance');
    }
  }
  
  // طريقة للسحب
  void withdraw(double amount) {
    if (amount > 0 && amount <= _balance) {
      _balance -= amount;
      print('Withdrew $amount. New balance: $_balance');
    } else {
      print('Invalid withdrawal amount');
    }
  }
}

void main() {
  var account = BankAccount('123456', 1000);
  print(account.balance); // استخدام getter
  account.balance = 1500; // استخدام setter
  account.deposit(500);
  account.withdraw(200);
}

متى نستخدم Setters/Getters؟

يجب استخدام Setters/Getters عندما تحتاج إلى:

  • التحقق من صحة القيم قبل تعيينها
  • تنفيذ عمليات حسابية عند قراءة/كتابة القيم
  • توفير وصول مقيد إلى الخصائص الداخلية

الـ Static Members

مفهوم الـ Static

الـ Static members (خصائص أو دوال) تنتمي إلى الـ Class نفسه وليس إلى الكائنات المنشأة منه. يمكن الوصول إليها مباشرة من الـ Class دون الحاجة إلى إنشاء كائن.

مثال عملي 


class MathUtils {
  // Static property
  static const double pi = 3.14159;
  
  // Static method
  static double calculateCircleArea(double radius) {
    return pi * radius * radius;
  }
  
  // Static factory constructor
  static MathUtils getInstance() {
    return MathUtils();
  }
}

void main() {
  // استخدام static property
  print(MathUtils.pi);
  
  // استخدام static method
  var area = MathUtils.calculateCircleArea(5);
  print('Area: $area');
  
  // استخدام static factory constructor
  var utils = MathUtils.getInstance();
}

متى نستخدم Static؟

استخدم الـ Static عندما:

  • تحتاج إلى خصائص أو دوال مشتركة بين جميع الكائنات من نفس الـ Class
  • تريد إنشاء دوال مساعدة (utility functions) لا تحتاج إلى كائن
  • تريد تنفيذ أنماط تصميم مثل Singleton

الـ Mixins

مفهوم Mixins

الـ Mixins هي طريقة لإعادة استخدام كود من عدة تسلسلات هرمية للفئات. تسمح لك بإضافة وظائف إلى فئة دون استخدام الوراثة. في Dart، يتم تعريف Mixin باستخدام الكلمة المفتاحية mixin.

مثال عملي 


mixin Swimmer {
  void swim() {
    print('Swimming!');
  }
}

mixin Flyer {
  void fly() {
    print('Flying!');
  }
}

class Animal {
  String name;
  Animal(this.name);
}

class Duck extends Animal with Swimmer, Flyer {
  Duck(String name) : super(name);
}

void main() {
  var duck = Duck('Donald');
  duck.swim();
  duck.fly();
}

الفرق بين Mixins والوراثة

الـ Mixins تختلف عن الوراثة في أن:

  • يمكن تطبيق عدة Mixins على فئة واحدة
  • لا تحتوي Mixins على Constructors
  • لا تنشئ Mixins علاقة "is-a" مثل الوراثة
  • تستخدم عندما تحتاج إلى إضافة سلوكيات وليس عندما تحتاج إلى تمثيل علاقة هرمية

الـ Enums

مفهوم Enums

الـ Enum (اختصار لـ Enumeration) هو نوع خاص يمثل مجموعة ثابتة من القيم. يساعد في جعل الكود أكثر قراءة وأمانًا عن طريق تقليل استخدام القيم النصية أو الرقمية مباشرة.

مثال عملي 


// تعريف Enum بسيط
enum Day {
  sunday,
  monday,
  tuesday,
  wednesday,
  thursday,
  friday,
  saturday
}

// Enum مع دوال
enum Status {
  pending,
  approved,
  rejected;
  
  // دالة في Enum
  String get description {
    switch (this) {
      case Status.pending:
        return 'Your request is pending';
      case Status.approved:
        return 'Your request has been approved';
      case Status.rejected:
        return 'Your request has been rejected';
    }
  }
}

void main() {
  // استخدام Enum
  var today = Day.monday;
  print(today); // Output: Day.monday
  
  // استخدام Enum مع دوال
  var status = Status.approved;
  print(status.description); // Output: Your request has been approved
}

متى نستخدم Enums؟

استخدم Enums عندما:

  • لديك مجموعة محددة ومعروفة مسبقًا من القيم
  • تريد تجنب استخدام القيم النصية أو الرقمية مباشرة
  • تريد جعل الكود أكثر أمانًا ضد الأخطاء الكتابية
  • تريد إضافة دوال أو سلوكيات خاصة بالقيم

التاسكات والواجبات

المهمة 1: إنشاء Class للطلاب

قم بإنشاء Class اسمه Student يحتوي على:

  • خصائص: name (String), id (int), grades (List<double>)
  • Constructor يأخذ name و id
  • دالة addGrade لإضافة درجة إلى List grades
  • دالة calculateAverage لحساب متوسط الدرجات
  • Getter للحصول على أعلى درجة

المهمة 2: تطبيق الوراثة

قم بإنشاء Class اسمه Person يحتوي على name و age، ثم أنشئ Class اسمه Employee يرث من Person ويضيف:

  • خاصية salary
  • دالة increaseSalary تزيد الراتب بنسبة معينة
  • Override لدالة toString لطباعة معلومات الموظف

المهمة 3: تطبيق Polymorphism

قم بإنشاء Abstract Class اسمه Shape بدالة calculateArea، ثم أنشئ Classين يرثان منه:

  • Circle مع خاصية radius وتنفيذ calculateArea
  • Rectangle مع خصائص width و height وتنفيذ calculateArea

أنشئ List من Shape وأضف إليه كائنات من النوعين، ثم استدع calculateArea لكل كائن.

المهمة 4: تطبيق Mixins

قم بإنشاء Mixinين:

  • Logger بدالة log تطبع رسالة
  • Timestamp بدالة getCurrentTime تعيد الوقت الحالي

ثم أنشئ Class يستخدم كلا الـ Mixinين ويجمع بين وظائفهما.

المهمة 5: تطبيق Enums

قم بإنشاء Enum اسمه UserRole يحتوي على:

  • admin, editor, viewer
  • دالة getPermissions تعيد قائمة صلاحيات كل دور

أنشئ Class User يستخدم هذا Enum.
