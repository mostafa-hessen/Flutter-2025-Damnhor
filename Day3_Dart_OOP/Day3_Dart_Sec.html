مقدمة عن البرمجة الكائنية (OOP)

البرمجة الكائنية (Object-Oriented Programming - OOP) هي نموذج برمجة يعتمد على مفهوم "الكائنات" التي تحتوي على بيانات وتعليمات برمجية. البيانات تمثل خصائص الكائن، والتعليمات البرمجية تمثل السلوك أو ما يمكن للكائن القيام به.

لماذا نستخدم OOP؟

  • تنظيم الكود بشكل أفضل وجعله أكثر قابلية للفهم
  • إعادة استخدام الكود (Code Reusability)
  • تسهيل الصيانة والتعديل
  • إخفاء البيانات (Data Hiding) لزيادة الأمان
  • نمذجة العالم الحقيقي بشكل أكثر واقعية

المفاهيم الأساسية في OOP

  • Class (الصنف): نموذج أو قالب لإنشاء الكائنات، يحدد الخصائص والسلوكيات
  • Object (الكائن): نسخة من الصنف، يحتوي على بيانات حقيقية
  • Inheritance (الوراثة): قدرة صنف على اكتساب خصائص وسلوكيات صنف آخر
  • Polymorphism (تعدد الأشكال): قدرة الكائنات على الاستجابة بطرق مختلفة لنفس الرسالة
  • Encapsulation (التغليف): إخفاء تفاصيل التنفيذ الداخلية عن المستخدم
  • Abstraction (التجريد): عرض الوظائف الأساسية فقط وإخفاء التفاصيل

ملاحظة مهمة

Dart هي لغة كائنية التوجه بالكامل، حتى الأنواع الأساسية مثل الأرقام والنصوص هي كائنات في Dart.

Class و Object في Dart

الصنف (Class) هو نموذج أو مخطط لإنشاء الكائنات. لنبدأ بمثال بسيط عن صنف Person:

class Person {
  // خصائص الصنف (Class Properties)
  String name;
  int age;
  
  // دالة لعرض المعلومات (Method)
  void displayInfo() {
    print('Name: $name, Age: $age');
  }
}

// إنشاء كائن من الصنف Person
void main() {
  // إنشاء كائن person1
  Person person1 = Person();
  
  // تعيين قيم للخصائص
  person1.name = 'أحمد';
  person1.age = 25;
  
  // استدعاء الدالة
  person1.displayInfo(); // الناتج: Name: أحمد, Age: 25
}

شرح المثال

  • قمنا بتعريف صنف Person يحتوي على خاصيتين: name و age
  • أضفنا دالة displayInfo() لعرض معلومات الشخص
  • في الدالة main()، أنشأنا كائنًا من الصنف Person باستخدام Person()
  • قمنا بتعيين قيم للخصائص ثم استدعينا الدالة لعرض المعلومات

نصيحة عملية

تخيل الصنف كقالب لصنع الكعك، والكائنات كقطع الكعك الفعلية. القالب يحدد الشكل والخصائص، ولكن كل قطعة كعك لها قيمها الخاصة (مثل اللون، الطعم، الحجم).

الـ Constructors بأنواعها

البناء (Constructor) هو دالة خاصة تُستدعى تلقائيًا عند إنشاء كائن من الصنف. يستخدم لتهيئة الكائن.

1. البناء الأساسي (Default Constructor)

class Person {
  String name;
  int age;
  
  // البناء الأساسي
  Person(String name, int age) {
    this.name = name;
    this.age = age;
  }
  
  // يمكن اختصاره إلى:
  Person(this.name, this.age);
}

2. البناء المسمى (Named Constructor)

يسمح لنا بتعريف أكثر من بناء في نفس الصنف لأغراض مختلفة.

class Person {
  String name;
  int age;
  
  // البناء الأساسي
  Person(this.name, this.age);
  
  // بناء مسمى لإنشاء شخص بدون اسم
  Person.unnamed(this.age) {
    name = 'غير معروف';
  }
  
  // بناء مسمى لإنشاء شخص حديث الولادة
  Person.newborn() {
    name = 'حديث الولادة';
    age = 0;
  }
}

void main() {
  Person p1 = Person('أحمد', 25);
  Person p2 = Person.unnamed(30);
  Person p3 = Person.newborn();
}

3. البناء مع المعاملات المسماة (Named Parameters)

يسمح بتمرير المعاملات بأي ترتيب مع تحديد أسماءها.

class Person {
  String name;
  int age;
  
  // بناء مع معاملات مسماة
  Person({required this.name, required this.age});
  
  // بناء مع معاملات مسماة اختيارية
  Person.optional({this.name = 'غير معروف', this.age = 0});
}

void main() {
  // يجب تحديد اسم المعامل عند الاستخدام
  Person p1 = Person(name: 'أحمد', age: 25);
  Person p2 = Person.optional(); // سيستخدم القيم الافتراضية
}

تمرين عملي

قم بإنشاء صنف Book يحتوي على الخصائص التالية: العنوان، المؤلف، سنة النشر، السعر. ثم:

  1. أنشئ بناءً أساسيًا يأخذ جميع الخصائص
  2. أنشئ بناءً مسمىً لإنشاء كتاب مجاني (سعره صفر)
  3. أنشئ بناءً مع معاملات مسماة
  4. أنشئ دالة لعرض معلومات الكتاب
  5. أنشئ عدة كائنات من الصنف Book باستخدام أنواع البناء المختلفة

مفهوم الـ Inheritance والـ Super

الوراثة (Inheritance) تسمح لصنف بكتساب خصائص وسلوكيات صنف آخر. الصنف الذي يرث يسمى صنف فرعي (Subclass)، والصنف الموروث منه يسمى صنف أب (Superclass).

class Animal {
  String name;
  int age;
  
  Animal(this.name, this.age);
  
  void eat() {
    print('$name يأكل');
  }
  
  void sleep() {
    print('$name ينام');
  }
}

class Dog extends Animal {
  String breed;
  
  // استخدام super لاستدعاء بناء الصنف الأب
  Dog(String name, int age, this.breed) : super(name, age);
  
  void bark() {
    print('$name ينبح');
  }
  
  // تجاوز (override) لدالة من الصنف الأب
  @override
  void eat() {
    print('$name يأكل بسرعة!');
    super.eat(); // استدعاء الدالة الأصلية من الصنف الأب
  }
}

void main() {
  Dog myDog = Dog('ريكس', 3, 'جولدن ريتريفر');
  myDog.eat();   // من الصنف Dog
  myDog.sleep(); // من الصنف Animal
  myDog.bark();  // من الصنف Dog
}

كلمة super

الكلمة الأساسية super تُستخدم للوصول إلى أعضاء الصنف الأب:

  • في البناء: لاستدعاء بناء الصنف الأب
  • في الدوال: لاستدعاء دوال الصنف الأب عند تجاوزها

ملاحظة هامة

Dart تدور الوراثة المفردة فقط، أي أن الصنف يمكن أن يرث من صنف أب واحد فقط، لكن يمكن تنفيذ عدة واجهات.

الـ Polymorphism والـ Override

تعدد الأشكال (Polymorphism) يعني قدرة الكائنات على الاستجابة بطرق مختلفة لنفس الرسالة (استدعاء الدالة).

class Shape {
  void draw() {
    print('رسم شكل عام');
  }
}

class Circle extends Shape {
  @override
  void draw() {
    print('رسم دائرة');
  }
}

class Square extends Shape {
  @override
  void draw() {
    print('رسم مربع');
  }
}

void main() {
  List<Shape> shapes = [Shape(), Circle(), Square()];
  
  for (var shape in shapes) {
    shape.draw(); // ستستجيب كل كائن بطريقة مختلفة
  }
  
  /* الناتج:
  رسم شكل عام
  رسم دائرة
  رسم مربع
  */
}

شرح المثال

  • قمنا بتعريف صنف Shape بدالة draw()
  • أنشأنا صنفين فرعيين Circle و Square وقمنا بتجاوز دالة draw() في كل منهما
  • في الدالة main()، أنشأنا قائمة تحتوي على كائنات من الأنواع المختلفة
  • عند استدعاء draw() على كل كائن، تم تنفيذ الدالة المناسبة لنوع الكائن

نصيحة عملية

تعدد الأشكال يسمح بكتابة كود أكثر عمومية ومرونة. يمكنك معالجة كائنات مختلفة بنفس الطريقة مع الحصول على سلوك خاص بكل كائن.

الـ Abstract Classes والـ Interfaces

1. الفئات المجردة (Abstract Classes)

الفئة المجردة هي فئة لا يمكن إنشاء كائنات منها مباشرة، وتُستخدم كقاعدة للفئات الأخرى.

// تعريف فئة مجردة
abstract class Vehicle {
  // دالة مجردة (بدون تنفيذ)
  void start();
  
  // دالة عادية (يمكن أن تحتوي على تنفيذ)
  void stop() {
    print('المركبة توقفت');
  }
}

class Car extends Vehicle {
  @override
  void start() {
    print('السيارة تعمل بالمحرك');
  }
}

class Bicycle extends Vehicle {
  @override
  void start() {
    print('الدراجة تعمل بالدواسات');
  }
}

void main() {
  // Vehicle v = Vehicle(); // خطأ: لا يمكن إنشاء كائن من فئة مجردة
  Car car = Car();
  car.start(); // السيارة تعمل بالمحرك
  car.stop();  // المركبة توقفت
  
  Bicycle bike = Bicycle();
  bike.start(); // الدراجة تعمل بالدواسات
}

2. الواجهات (Interfaces)

في Dart، كل فئة تعتبر واجهة بشكل ضمني. يمكن استخدام الكلمة الأساسية implements لتنفيذ واجهة فئة أخرى.

class Logger {
  void log(String message) => print(message);
}

class ConsoleLogger implements Logger {
  @override
  void log(String message) {
    print('[CONSOLE] $message');
  }
}

class FileLogger implements Logger {
  @override
  void log(String message) {
    print('[FILE] $message');
  }
}

void main() {
  Logger logger1 = ConsoleLogger();
  logger1.log('رسالة تجريبية'); // [CONSOLE] رسالة تجريبية
  
  Logger logger2 = FileLogger();
  logger2.log('رسالة أخرى'); // [FILE] رسالة أخرى
}

الفرق بين extends و implements

extends implements
يورث التنفيذ من الفئة الأب يجب تنفيذ جميع الأعضاء الموروثة
يمكن استخدام super للوصول لأعضاء الفئة الأب لا يمكن استخدام super
يمكن أن يورث من فئة واحدة فقط يمكن تنفيذ عدة واجهات
يستخدم للعلاقة "هو" (is-a) يستخدم للعلاقة "يمكنه" (can-do)

الـ Encapsulation والـ Setters/Getters

التغليف (Encapsulation) هو مبدأ إخفاء تفاصيل التنفيذ الداخلية للكائن عن المستخدم، والسماح بالوصول للبيانات فقط عبر واجهة محددة.

class BankAccount {
  // خصائص خاصة (تبدأ بشرطة سفلية)
  String _accountNumber;
  double _balance;
  
  BankAccount(this._accountNumber, this._balance);
  
  // getter للرصيد
  double get balance => _balance;
  
  // setter للرصيد (مع التحقق من القيمة)
  set balance(double value) {
    if (value >= 0) {
      _balance = value;
    } else {
      print('لا يمكن أن يكون الرصيد سالبًا');
    }
  }
  
  // دالة للإيداع
  void deposit(double amount) {
    if (amount > 0) {
      _balance += amount;
      print('تم إيداع $amount، الرصيد الحالي: $_balance');
    } else {
      print('المبلغ يجب أن يكون موجبًا');
    }
  }
  
  // دالة للسحب
  void withdraw(double amount) {
    if (amount > 0 && amount <= _balance) {
      _balance -= amount;
      print('تم سحب $amount، الرصيد الحالي: $_balance');
    } else {
      print('المبلغ غير صالح أو يتجاوز الرصيد');
    }
  }
}

void main() {
  BankAccount account = BankAccount('123456', 1000);
  
  // account._balance = -500; // خطأ: الخاصية خاصة
  print(account.balance); // 1000.0 (عن طريق getter)
  
  account.balance = 1500; // عن طريق setter
  account.balance = -200; // سيطبع رسالة الخطأ
  
  account.deposit(300);
  account.withdraw(2000);
  account.withdraw(500);
}

فوائد التغليف

  • إخفاء تفاصيل التنفيذ الداخلية
  • التحكم في كيفية الوصول إلى البيانات وتعديلها
  • إضافة شروط التحقق عند تعديل البيانات
  • جعل الكود أكثر أمانًا وسهولة في الصيانة

ملاحظة

في Dart، لا توجد كلمة رئيسية مثل private، ولكن أي عضو يبدأ بشرطة سفلية _ يعتبر خاصًا بالملف (file-private) ولا يمكن الوصول إليه من ملفات أخرى.

الـ Static Members

الأعضاء الثابتة (Static Members) تنتمي للصنف نفسه وليس للكائنات المنشأة منه. يمكن الوصول إليها مباشرة من اسم الصنف.

class MathUtils {
  // ثابت (static constant)
  static const double pi = 3.14159;
  
  // متغير ثابت (static variable)
  static int usageCount = 0;
  
  // دالة ثابتة (static method)
  static double calculateCircleArea(double radius) {
    usageCount++;
    return pi * radius * radius;
  }
}

void main() {
  // لا نحتاج إلى إنشاء كائن لاستخدام الأعضاء الثابتة
  print(MathUtils.pi); // 3.14159
  
  double area = MathUtils.calculateCircleArea(5);
  print('المساحة: $area'); // المساحة: 78.53975
  print('عدد الاستخدامات: ${MathUtils.usageCount}'); // 1
  
  MathUtils.calculateCircleArea(3);
  print('عدد الاستخدامات: ${MathUtils.usageCount}'); // 2
}

متى نستخدم الأعضاء الثابتة؟

  • لإنشاء دوال مساعدة (utility functions) لا تحتاج إلى حالة كائن
  • لتخزين بيانات مشتركة بين جميع كائنات الصنف
  • لتعريف الثوابت المرتبطة بالصنف
  • لتنفيذ أنماط التصميم مثل Singleton

تمرين عملي

قم بإنشاء صنف Counter يحتوي على:

  1. متغير ثابت totalCount يحسب عدد مرات إنشاء كائنات من هذا الصنف
  2. متغير ثابت maxCount يحدد الحد الأقصى لعدد الكائنات المسموح بها (10)
  3. في البناء، قم بزيادة totalCount والتحقق من عدم تجاوز maxCount
  4. أضف دالة ثابتة resetCount() لإعادة العداد إلى الصفر
  5. جرب إنشاء عدة كائنات وطباعة totalCount بعد كل إنشاء

الـ Mixins

Mixins هي طريقة لإعادة استخدام شفرة من عدة تسلسلات هرمية للفئات. تسمح بإضافة وظائف إلى فئة بدون استخدام الوراثة.

// تعريف mixin
mixin LoggerMixin {
  void log(String message) {
    print('[LOG] ${DateTime.now()}: $message');
  }
}

mixin JsonConverterMixin {
  String toJson() {
    return '{}';
  }
}

// استخدام mixins مع فئة
class User with LoggerMixin, JsonConverterMixin {
  String name;
  int age;
  
  User(this.name, this.age);
  
  void displayInfo() {
    log('عرض معلومات المستخدم'); // من LoggerMixin
    print('Name: $name, Age: $age');
  }
  
  @override
  String toJson() { // من JsonConverterMixin
    return '{"name": "$name", "age": $age}';
  }
}

void main() {
  User user = User('أحمد', 25);
  user.displayInfo();
  print(user.toJson());
  
  /* الناتج:
  [LOG] 2023-05-15 10:30:45.123: عرض معلومات المستخدم
  Name: أحمد, Age: 25
  {"name": "أحمد", "age": 25}
  */
}

الفرق بين mixins والوراثة

  • Mixins تسمح بإعادة استخدام الكود من عدة مصادر، بينما الوراثة من مصدر واحد فقط
  • Mixins لا تنشئ علاقة "is-a" مثل الوراثة
  • يمكن استخدام عدة mixins مع فئة واحدة
  • Mixins لا يمكن إنشاء كائنات منها مباشرة

ملاحظة هامة

يمكن تحديد أن mixin معينة يمكن استخدامها فقط مع فئات معينة باستخدام الكلمة الأساسية on:

mixin SpecialMixin on SomeBaseClass {
  // الكود هنا
}

الـ Enums

التعدادات (Enums) هي نوع خاص يستخدم لتعريف مجموعة ثابتة من القيم ذات الصلة.

// تعريف enum
enum Day {
  saturday,
  sunday,
  monday,
  tuesday,
  wednesday,
  thursday,
  friday
}

// enum مع قيم مخصصة
enum Status {
  pending('في الانتظار'),
  approved('مقبول'),
  rejected('مرفوض');
  
  final String arabicName;
  const Status(this.arabicName);
}

void main() {
  // استخدام enum
  Day today = Day.monday;
  print(today); // Day.monday
  print(today.index); // 2 (الترتيب الصفري)
  
  // استخدام enum مع قيم مخصصة
  Status myStatus = Status.approved;
  print(myStatus.arabicName); // مقبول
  
  // التكرار على جميع قيم enum
  for (var day in Day.values) {
    print(day);
  }
  
  // استخدام switch مع enum
  switch (today) {
    case Day.saturday:
    case Day.sunday:
      print('عطلة نهاية الأسبوع');
      break;
    case Day.friday:
      print('يوم الجمعة');
      break;
    default:
      print('يوم عمل');
  }
}

فوائد استخدام Enums

  • تحسين قابلية قراءة الكود
  • تقليل الأخطاء باستخدام قيم محددة مسبقًا
  • سهولة في الصيانة عند إضافة أو تعديل القيم
  • التكامل الجيد مع تعليمات التحكم مثل switch

نصيحة عملية

استخدم Enums عندما يكون لديك مجموعة محددة ومعروفة مسبقًا من القيم المتعلقة ببعضها، مثل أيام الأسبوع، حالات الطلب، أنواع المستخدمين، إلخ.
