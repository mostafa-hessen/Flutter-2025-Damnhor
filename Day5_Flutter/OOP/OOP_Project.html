البرمجة الكائنية (OOP) - مشروع مصنع سيارات

تعلم مفاهيم OOP من خلال مثال عملي بلغة Dart

مقدمة

في هذا الدرس، سنتعلم مفاهيم البرمجة الكائنية (OOP) من خلال مشروع عملي لإنشاء نظام إدارة لمصنع سيارات. سنبدأ بالبرمجة الإجرائية ثم ننتقل تدريجياً إلى OOP لنرى الفرق.

الهدف: فهم الفرق بين البرمجة الإجرائية والكائنية وتطبيق مبادئ OOP الأساسية.

البرمجة الإجرائية (Procedural Programming)

في البرمجة الإجرائية، ننظم الكود كمجموعة من الإجراءات (الدوال) التي تعمل على بيانات منفصلة.

مثال: إدارة سيارات المصنع

// بيانات السيارات في Lists منفصلة
List<String> brands = [];
List<String> models = [];
List<int> years = [];
List<int> speeds = [];

// دالة لإضافة سيارة جديدة
void addCar(String brand, String model, int year) {
  brands.add(brand);
  models.add(model);
  years.add(year);
  speeds.add(0); // السرعة الافتراضية صفر
}

// دالة لتسريع السيارة
void accelerate(int index, int amount) {
  speeds[index] += amount;
}

// دالة لعرض بيانات السيارة
void display(int index) {
  print('Brand: ${brands[index]}');
  print('Model: ${models[index]}');
  print('Year: ${years[index]}');
  print('Speed: ${speeds[index]} km/h');
}

مشاكل البرمجة الإجرائية

العيب الشرح
❌ البيانات متبعثرة كل خاصية موجودة في List مختلفة، ولازم دايمًا المؤشر `index` يكون متطابق بينهم
❌ قابلية التوسع ضعيفة لو حبينا نضيف خاصية زي "لون العربية" لازم نعدل كل القوائم ودوال التعامل معاها
❌ صعب نعيد استخدام الكود مفيش مفهوم الكائن (Object) فنحتاج نكتب دوال منفصلة لكل كيان
❌ التحكم في البيانات محدود أي كود ممكن يغير السرعة مباشرة، مفيش حماية أو شروط (Encapsulation)

الحل: البرمجة الكائنية (OOP)

البرمجة الكائنية تنظم الكود في كائنات (Objects) تحتوي على البيانات والسلوكيات معاً، مما يجعل الكود أكثر تنظيماً وقابلية للصيانة والتوسع.

المشكلة في البرمجة الإجرائية

  • ❌ بيانات متفرقة
  • ❌ مفيش تنظيم
  • ❌ صعب نضيف خصائص
  • ❌ مش آمن

الحل في البرمجة الكائنية

  • ✅ بيانات داخل كائن واحد `Car`
  • ✅ كلاس منظم بخصائص ودوال
  • ✅ نضيف خصائص داخل الكلاس فقط
  • ✅ نستخدم `private` و`getters/setters`

خطوات تطبيق OOP

1

إنشاء كلاس أساسي للسيارة

نبدأ بإنشاء كلاس `Car` يحتوي على الخصائص الأساسية للسيارة والدوال التي تعمل عليها.

class Car {
  String brand;
  String model;
  int year;
  int speed = 0;

  Car(this.brand, this.model, this.year);

  void accelerate(int amount) {
    speed += amount;
  }

  void display() {
    print('Brand: $brand');
    print('Model: $model');
    print('Year: $year');
    print('Speed: $speed km/h');
  }
}
2

استخدام Abstract Class لتحديد المواصفات العامة

نستخدم `abstract class` لتحديد السلوكيات المشتركة التي يجب أن تنفذها جميع المركبات.

abstract class Vehicle {
  void start();
  void accelerate(int speed);
}

لماذا نستخدم Abstract Class؟

  • تحديد السلوكيات المشتركة (كل العربيات عندها start و accelerate)
  • فرض التنفيذ على الكلاسات التي تطبق هذا الـ interface
  • لا يمكن إنشاء كائنات مباشرة من الـ abstract class
3

تطبيق الـ Interface على كلاس Car

نجعل كلاس `Car` يطبق الـ `Vehicle` مما يفرض عليه تنفيذ الدوال المطلوبة.

class Car implements Vehicle {
  String brand;
  String model;
  int year;
  int _speed = 0;

  Car(this.brand, this.model, this.year);

  @override
  void start() {
    print('$brand $model is starting...');
  }

  @override
  void accelerate(int speed) {
    _speed += speed;
    print('$brand $model accelerated to $_speed km/h');
  }

  void displayInfo() {
    print('Brand: $brand');
    print('Model: $model');
    print('Year: $year');
    print('Speed: $_speed');
  }
}
4

إنشاء أنواع متخصصة من السيارات (Inheritance)

نستخدم الوراثة لإنشاء أنواع متخصصة من السيارات مع تعديل السلوكيات حسب الحاجة.

GasCar (سيارة بنزين)

class GasCar extends Car {
  GasCar(String brand, String model, int year) 
      : super(brand, model, year);

  @override
  void start() {
    print('$brand $model (Gas) starting with engine sound...');
  }

  @override
  void accelerate(int speed) {
    super.accelerate(speed);
    print('$brand (Gas) engine roaring to $speed km/h!');
  }
}

ElectricCar (سيارة كهربائية)

class ElectricCar extends Car {
  ElectricCar(String brand, String model, int year) 
      : super(brand, model, year);

  @override
  void start() {
    print('$brand $model (Electric) starting silently...');
  }

  @override
  void accelerate(int speed) {
    super.accelerate(speed);
    print('$brand (Electric) glides quietly to $speed km/h.');
  }
}

Polymorphism في العمل

يمكننا معاملة جميع السيارات ككائنات من نوع `Vehicle` مع تنفيذ مختلف للدوال حسب النوع.

5

إضافة Encapsulation و Static

نحمي البيانات ونضيف متغيرات static لحساب عدد السيارات المصنوعة.

class Car implements Vehicle {
  String brand;
  String model;
  int year;
  int _speed = 0; // private speed
  String _type = "Unknown"; // private type
  
  static int carsCount = 0; // static counter

  Car(this.brand, this.model, this.year) {
    carsCount++; // زيادة العداد عند الإنشاء
  }

  // Getter للسرعة
  int get speed => _speed;

  // Method آمنة لتغيير السرعة
  void increaseSpeed(int value) {
    if (value > 0) {
      _speed += value;
    }
  }

  @override
  void accelerate(int speed) {
    increaseSpeed(speed);
    print('Accelerated to $_speed km/h');
  }
  
  // باقي الدوال...
}

Encapsulation

  • جعلنا `_speed` و `_type` private باستخدام _
  • وفرنا getter للقراءة فقط
  • أنشأنا method آمنة لتعديل السرعة

Static

  • `carsCount` يحسب جميع السيارات المصنوعة
  • يتشاركه جميع الكائنات من Car وأولادها
  • يمكن الوصول إليه مباشرة من الكلاس

الكود النهائي والتجربة

الكود الكامل

abstract class Vehicle {
  void start();
  void accelerate(int speed);
}

class Car implements Vehicle {
  String brand;
  String model;
  int year;
  int _speed = 0;
  String _type = "Unknown";
  static int carsCount = 0;

  Car(this.brand, this.model, this.year) {
    carsCount++;
  }

  int get speed => _speed;
  String get type => _type;

  void increaseSpeed(int value) {
    if (value > 0) {
      _speed += value;
    }
  }

  @override
  void start() {
    print('$brand $model is starting...');
  }

  @override
  void accelerate(int speed) {
    increaseSpeed(speed);
    print('$brand $model accelerated to $_speed km/h');
  }

  void displayInfo() {
    print('Brand: $brand');
    print('Model: $model');
    print('Year: $year');
    print('Speed: $_speed');
    print('Type: $_type');
  }
}

class GasCar extends Car {
  GasCar(String brand, String model, int year) : super(brand, model, year) {
    _type = "Gas";
  }

  @override
  void start() {
    print('$brand $model (Gas) is starting with engine sound...');
  }

  @override
  void accelerate(int speed) {
    increaseSpeed(speed);
    print('$brand (Gas) engine roaring to $speed km/h!');
  }
}

class ElectricCar extends Car {
  ElectricCar(String brand, String model, int year) : super(brand, model, year) {
    _type = "Electric";
  }

  @override
  void start() {
    print('$brand $model (Electric) is starting silently...');
  }

  @override
  void accelerate(int speed) {
    increaseSpeed(speed);
    print('$brand (Electric) glides quietly to $speed km/h.');
  }
}

void main() {
  GasCar gasCar = GasCar("Toyota", "Corolla", 2022);
  ElectricCar electricCar = ElectricCar("Tesla", "Model 3", 2023);
  GasCar gasCar2 = GasCar("Honda", "Civic", 2021);

  gasCar.start();
  gasCar.accelerate(60);
  gasCar.displayInfo();

  print('-----');

  electricCar.start();
  electricCar.accelerate(80);
  electricCar.displayInfo();

  print('-----');
  
  print('Total cars produced: ${Car.carsCount}');
}

ملخص مبادئ OOP المطبقة

Class & Object

عرفنا كلاسات `Car`, `GasCar`, `ElectricCar` وأنشأنا كائنات منها.

Encapsulation

جعلنا `_speed` و `_type` private ووفرنا methods آمنة للوصول إليها.

Inheritance

`GasCar` و `ElectricCar` ورثوا من `Car` واستخدموا `super` للوصول للأب.

Polymorphism

كل عربية لها تنفيذ مختلف لـ `start()` و `accelerate()` حسب نوعها.

Abstraction

`Vehicle` abstract class حددت المواصفات العامة دون تنفيذ.

Static

`carsCount` يحسب جميع السيارات المصنوعة ويشارك بين الكائنات.
