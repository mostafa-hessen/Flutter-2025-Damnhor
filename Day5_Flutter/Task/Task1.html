تاسك مشروع: نظام إدارة الموظفين والشركات

تطبيق مبادئ البرمجة الكائنية (OOP) في مشروع عملي

Encapsulation Inheritance Polymorphism Abstraction

الهدف من المشروع

تطبيق المبادئ الأساسية للبرمجة الكائنية (OOP) في نظام لإدارة الموظفين والشركات، مع التركيز على:

  • إنشاء هيكل منظم للكائنات والعلاقات بينها
  • تطبيق مبادئ التغليف، الوراثة، تعدد الأشكال والتجريد
  • بناء نظام قابل للتوسع والصيانة
1

إنشاء الكلاس المجرد (Abstract Class)

  • • إنشاء كلاس Employee كـ abstract
  • • إضافة خصائص خاصة _name و _salary
  • • توفير getters و setters للتحكم في الخصائص
  • • إضافة دالة abstract work()
  • • إضافة دالة showInfo() لعرض البيانات
2

إنشاء الفئات الفرعية (Inheritance)

كلاس Developer:

  • • إضافة خاصية programmingLanguage
  • • تعديل دالة work() لتتناسب مع المطور

كلاس Manager:

  • • إضافة خاصية department
  • • إضافة دالة manage() خاصة بالمدير
3

إنشاء كلاس الشركة (Company Class)

  • • إضافة خاصية name للشركة
  • • قائمة employees لتخزين الموظفين
  • • دالة addEmployee() لإضافة موظف
  • • دالة showAllEmployees() لعرض الجميع
4

التنفيذ في دالة main

  • • إنشاء كائنات من نوع Company
  • • إضافة موظفين (مطورين ومديرين)
  • • عرض معلومات الموظفين وسلوكياتهم
  • • تجربة تعديل الرواتب عبر setter

معايير التقييم

التغليف (Encapsulation)

  • استخدام خصائص خاصة
  • توفير واجهات تحكم (getters/setters)
  • حماية البيانات من التعديل المباشر

الوراثة (Inheritance)

  • إنشاء هيكل وراثة صحيح
  • استخدام الكلمة المفتاحية extends
  • استدعاء منشئ الأب super

تعدد الأشكال (Polymorphism)

  • إعادة تعريف الدوال (override)
  • سلوك مختلف لكل فئة فرعية
  • استدعاء دوال الأب عند الحاجة

التجريد (Abstraction)

  • استخدام abstract class
  • وجود abstract methods
  • عدم إمكانية إنشاء كائن مباشر
