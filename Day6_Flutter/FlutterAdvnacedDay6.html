Container

الـ Container هو أحد أكثر الـ Widgetات استخدامًا في Flutter. يمكن اعتباره صندوقًا قابل للتخصيص يمكن أن يحتوي على Widgetات أخرى.

info الخصائص الرئيسية:

  • child: الـ Widget الذي سيحتويه
  • color: لون الخلفية
  • width/height: تحديد الأبعاد
  • margin/padding: تحديد الهوامش والحشو
  • decoration: إضافة حدود، تدرجات، ظلال

lightbulb متى تستخدم Container؟

  • عندما تحتاج إلى مساحة محددة الأبعاد

  • عندما تريد إضافة هامش حول Widget معين
  • كحاوية عامة لتجميع Widgetات أخرى

Image

Widget الـ Image يستخدم لعرض الصور في تطبيق Flutter. يمكن تحميل الصور من مصادر مختلفة.

info طرق تحميل الصور:

  • Image.asset: لتحميل الصور من مجلد assets في المشروع
  • Image.network: لتحميل الصور من الإنترنت
  • Image.file: لتحميل الصور من ملف في الجهاز
  • Image.memory: لتحميل الصور من بيانات بايت في الذاكرة
// مثال على تحميل صورة من الإنترنت
Image.network(
  'https://example.com/image.jpg',
  width: 200,
  height: 200,
  fit: BoxFit.cover,
)

// مثال على تحميل صورة من assets
Image.asset(
  'assets/images/logo.png',
  width: 100,
  height: 100,
)

// إضافة صورة مع placeholder أثناء التحميل
FadeInImage.assetNetwork(
  placeholder: 'assets/loading.gif',
  image: 'https://example.com/image.jpg',
)

Column و Row

الـ Column و Row هما Widgetات التخطيط الأساسية في Flutter لترتيب العناصر رأسيًا أو أفقيًا.

compare مقارنة بين Column و Row:

  • Column: ترتيب العناصر من الأعلى إلى الأسفل (رأسيًا)
  • Row: ترتيب العناصر من اليمين إلى اليسار (أفقيًا)
  • كلاهما يستخدم children لقائمة العناصر الفرعية
  • كلاهما يدعم خصائص المحاذاة والتوزيع (mainAxisAlignment و crossAxisAlignment)
// مثال على Column
Column(
  mainAxisAlignment: MainAxisAlignment.center,
  crossAxisAlignment: CrossAxisAlignment.stretch,
  children: [
    Text('عنصر 1'),
    Text('عنصر 2'),
    Text('عنصر 3'),
  ],
)

// مثال على Row
Row(
  mainAxisAlignment: MainAxisAlignment.spaceBetween,
  children: [
    Icon(Icons.star),
    Icon(Icons.star),
    Icon(Icons.star),
  ],
)

Stack

Widget الـ Stack يستخدم لترتيب العناصر فوق بعضها البعض (طبقات). يمكن التحكم في موقع كل عنصر باستخدام Positioned Widget.

info الخصائص الرئيسية:

  • children: قائمة العناصر المراد تكديسها
  • alignment: محاذاة العناصر (الافتراضي هو topStart)
  • fit: كيفية تكيف العناصر مع Stack
  • clipBehavior: كيفية قص العناصر التي تتجاوز حدود Stack
// مثال على Stack
Stack(
  children: [
    Container(
      width: 200,
      height: 200,
      color: Colors.blue,
    ),
    Positioned(
      bottom: 10,
      right: 10,
      child: Container(
        width: 100,
        height: 100,
        color: Colors.red,
      ),
    ),
    Positioned(
      top: 20,
      left: 20,
      child: Icon(Icons.star, size: 40, color: Colors.white),
    ),
  ],
)

ListTile في Flutter

الـ ListTile هو Widget متخصص لعرض صفوف في القوائم، يوفر تخطيطًا قياسيًا للعناصر مع أيقونة وعنوان ووصف فرعي.

info الخصائص الرئيسية

  • leading: عنصر في البداية (عادة أيقونة)
  • title: النص الرئيسي للعنصر
  • subtitle: النص الثانوي (اختياري)
  • trailing: عنصر في النهاية (عادة أيقونة)
  • onTap: دالة عند النقر على العنصر

star مميزات ListTile

  • تخطيط متوافق مع إرشادات Material Design
  • دعم جيد لإمكانية الوصول (Accessibility)
  • تفاعل افتراضي عند النقر (ripple effect)
  • سهولة في إنشاء قوائم متسقة
// أمثلة على استخدام ListTile
// 1. عنصر قائمة أساسي
ListTile(
leading: Icon(Icons.email),
title: Text('البريد الإلكتروني'),
subtitle: Text('user@example.com'),
trailing: Icon(Icons.arrow_forward),
onTap: () {
// تنفيذ إجراء عند النقر
},
)

// 2. قائمة إعدادات
Column(
children: [
ListTile(
leading: Icon(Icons.settings),
title: Text('الإعدادات'),
),
ListTile(
leading: Icon(Icons.notifications),
title: Text('الإشعارات'),
trailing: Switch(value: true, onChanged: (v) {}),
),
ListTile(
leading: Icon(Icons.help),
title: Text('المساعدة'),
),
],
)

info خصائص Wrap

  • direction

    تحديد اتجاه الالتفاف (أفقي أو رأسي)

  • spacing

    المسافة بين العناصر في الاتجاه الرئيسي

  • runSpacing

    المسافة بين الأسطر في الاتجاه المعاكس

  • alignment

    محاذاة العناصر في الاتجاه الرئيسي

  • runAlignment

    محاذاة الأسطر في الاتجاه المعاكس

code مثال تطبيقي 

Wrap(
  spacing: 8.0, // المسافة بين العناصر
  runSpacing: 4.0, // المسافة بين الأسطر
  children: [
    Chip(label: Text('Flutter')),
    Chip(label: Text('Dart')),
    Chip(label: Text('Android')),
    Chip(label: Text('iOS')),
    Chip(label: Text('Web')),
    Chip(label: Text('Desktop')),
    Chip(label: Text('Firebase')),
    Chip(label: Text('Material Design')),
  ],
)
Flutter Dart Android iOS Web Desktop Firebase Material Design

compare مقارنة بين Wrap و Row

Row

  • يعرض جميع العناصر في سطر واحد
  • إذا زادت العناصر عن المساحة، يتم اقتصاصها
  • مناسب عندما تعرف أن جميع العناصر ستتناسب في سطر واحد
  • أداء أفضل من Wrap

Wrap

  • يلتف العناصر إلى أسطر جديدة عند الحاجة
  • لا يتم اقتصاص أي عناصر
  • مناسب للقوائم الديناميكية أو عندما لا تعرف عدد العناصر
  • أداء أقل قليلاً من Row

SingleChildScrollView

SingleChildScrollView هو Widget يسمح بالتمرير لمحتوى أكبر من المساحة المتاحة على الشاشة. يعتبر حلاً بسيطًا للتمرير عندما يكون لديك محتوى غير معقد.

info خصائص SingleChildScrollView

  • scrollDirection

    اتجاه التمرير (أفقي أو رأسي - الافتراضي رأسي)

  • reverse

    إذا كان true، يبدأ التمرير من النهاية

  • physics

    يحدد سلوك التمرير (مثل BouncingScrollPhysics)

  • padding

    حشو حول المحتوى

code مثال تطبيقي 

SingleChildScrollView(
  padding: EdgeInsets.all(16),
  child: Column(
    children: [
      Container(height: 200, color: Colors.blue),
      SizedBox(height: 20),
      Container(height: 200, color: Colors.red),
      SizedBox(height: 20),
      Container(height: 200, color: Colors.green),
      SizedBox(height: 20),
      Container(height: 200, color: Colors.yellow),
    ],
  ),
)

warning متى تستخدم SingleChildScrollView؟

check_circle حالات مناسبة

  • عندما يكون لديك عدد محدود من العناصر
  • عندما لا تحتاج إلى تحكم دقيق في أداء التمرير
  • للنماذج البسيطة أو المحتوى القصير نسبيًا

cancel حالات غير مناسبة

  • عندما يكون لديك قوائم طويلة أو ديناميكية (استخدم ListView بدلاً من ذلك)
  • عندما تحتاج إلى تحسين أداء التمرير
  • عندما تحتاج إلى ميزات متقدمة مثل التحميل البطيء (lazy loading)

Padding

Widget الـ Padding يضيف مساحة فارغة حول Widget آخر. يعتبر من أهم Widgetات التخطيط في Flutter.

info خصائص Padding

  • padding

    يحدد مقدار الحشو (يستخدم EdgeInsets)

  • child

    الـ Widget الذي سيتم تطبيق الحشو عليه

lightbulb EdgeInsets

  • EdgeInsets.all(16): حشو متساو من جميع الجهات
  • EdgeInsets.symmetric(vertical: 8, horizontal: 16): حشو متناظر
  • EdgeInsets.only(left: 8, top: 16): حشو من جهات محددة
  • EdgeInsets.fromLTRB(16, 8, 16, 8): حشو لكل جهة على حدة

code أمثلة على Padding 

// حشو متساو من جميع الجهات
Padding(
  padding: EdgeInsets.all(16),
  child: Text('نص مع حشو'),
)

// حشو متناظر
Padding(
  padding: EdgeInsets.symmetric(vertical: 8, horizontal: 16),
  child: Text('نص مع حشو رأسي وأفقي'),
)

// حشو من جهات محددة
Padding(
  padding: EdgeInsets.only(left: 8, top: 16),
  child: Text('نص مع حشو من اليسار والأعلى'),
)

compare الفرق بين Padding و Margin

Padding

  • يضيف مساحة داخل حدود الـ Widget
  • يؤثر على المساحة الداخلية للـ Widget
  • يتم تطبيقه باستخدام Widget Padding
  • يؤثر على لون الخلفية إذا كان للـ Widget خلفية

Margin

  • يضيف مساحة خارج حدود الـ Widget
  • يؤثر على المساحة بين الـ Widget والعناصر المحيطة
  • يتم تطبيقه عادةً باستخدام خاصية margin في Container
  • لا يؤثر على لون الخلفية

SizedBox

Widget الـ SizedBox يستخدم لإنشاء مساحة فارغة بحجم محدد أو لتحديد حجم ثابت لـ Widget آخر.

info استخدامات SizedBox

  • إنشاء مسافات بين العناصر

    بدلاً من استخدام Padding عندما تحتاج فقط إلى مسافة بسيطة

  • تحديد حجم ثابت لـ Widget

    لضمان أن يحتفظ Widget بحجم معين

  • إنشاء مساحة فارغة

    كعنصر نائب (placeholder) أثناء التحميل

lightbulb نصائح مهمة

  • استخدم SizedBox بدلاً من Container عندما تحتاج فقط إلى تحديد حجم
  • SizedBox أخف وزنًا من Container لأنه لا يحتوي على خصائص إضافية
  • يمكن استخدام SizedBox.shrink() لإنشاء مساحة فارغة بحجم صفر

code أمثلة على SizedBox 

// إنشاء مسافة بين عنصرين
Column(
  children: [
    Text('النص الأول'),
    SizedBox(height: 20), // مسافة 20 بكسل
    Text('النص الثاني'),
  ],
)

// تحديد حجم ثابت لصورة
SizedBox(
  width: 200,
  height: 200,
  child: Image.network('https://example.com/image.jpg'),
)

// إنشاء مساحة فارغة
SizedBox(width: 50, height: 50)

compare مقارنة بين SizedBox و Container

SizedBox

  • أبسط وأخف وزنًا
  • يستخدم فقط لتحديد الحجم أو إنشاء مسافات
  • لا يدعم الخصائص التصميمية (لون، حدود، إلخ)
  • أفضل للأداء عندما تحتاج فقط إلى تحديد حجم

Container

  • أكثر مرونة ويوفر خصائص تصميمية متعددة
  • يدعم الحشو، الهوامش، الألوان، الحدود، الظلال
  • أثقل قليلاً من SizedBox
  • يستخدم عندما تحتاج إلى أكثر من مجرد تحديد حجم

ThemeData في Flutter

الـ ThemeData هو كائن يحدد المظهر العام لتطبيق Flutter، حيث يحتوي على جميع الإعدادات المتعلقة بالألوان، الخطوط، الأشكال، وغيرها من الخصائص المرئية.

info الخصائص الرئيسية

  • colorScheme: مجموعة الألوان الأساسية للتطبيق
  • textTheme: أنماط النصوص المستخدمة في التطبيق
  • appBarTheme: إعدادات شريط التطبيق
  • buttonTheme: إعدادات الأزرار
  • inputDecorationTheme: إعدادات حقول الإدخال

)

Expanded في Flutter

الـ Expanded هو widget يستخدم داخل Column أو Row لملء المساحة المتاحة، حيث يأخذ الطفل المساحة المتبقية بناءً على عامل التمدد (flex factor).

info الخصائص الرئيسية

  • child: الـ Widget الذي سيتم توسيعه
  • flex: عامل التمدد (يحدد نسبة المساحة التي سيأخذها)

lightbulb حالات الاستخدام الشائعة

  • عندما تريد أن يأخذ عنصر مساحة أكبر من العناصر الأخرى
  • لتوزيع المساحة بين عدة عناصر بنسب مختلفة
  • لضمان استغلال كامل المساحة المتاحة
// أمثلة على استخدام Expanded
// 1. توزيع المساحة بالتساوي
Row(
children: [
Expanded(child: Container(color: Colors.red, height: 100)),
Expanded(child: Container(color: Colors.green, height: 100)),
Expanded(child: Container(color: Colors.blue, height: 100)),
],
)

// 2. توزيع المساحة بنسب مختلفة
Column(
children: [
Expanded(
flex: 2, // يأخذ ضعف المساحة
child: Container(color: Colors.amber),
),
Expanded(
flex: 1, // يأخذ نصف المساحة
child: Container(color: Colors.purple),
),
],
)

الفرق بين Expanded و Flexible

Expanded هو نوع من Flexible مع ضبط fit ليكون FlexFit.tight (يأخذ كل المساحة المتاحة)، بينما Flexible يمكن ضبطه ليكون FlexFit.loose (لا يأخذ كل المساحة بالضرورة).
