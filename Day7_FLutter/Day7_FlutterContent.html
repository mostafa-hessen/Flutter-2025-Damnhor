Stateful vs Stateless Widgets

في Flutter، هناك نوعان أساسيان من الـ Widgets: Stateless و Stateful. الفرق الرئيسي بينهما هو القدرة على التغيير مع الوقت.

toggle_off Stateless Widget

  • لا يتغير مع الوقت

    يتم رسمه مرة واحدة ولا يتغير إلا إذا تغيرت الـ props الممررة إليه

  • أمثلة

    Text, Icon, Container, وغيرها من الـ Widgetات الثابتة

  • الأداء

    أخف وأسرع لأنه لا يحتاج إلى إدارة حالة 

// مثال على Stateless Widget
class MyText extends StatelessWidget {
  final String text;
  
  const MyText({required this.text});
  
  @override
  Widget build(BuildContext context) {
    return Text(text);
  }
}

toggle_on Stateful Widget

  • يتغير مع الوقت

    يمكنه التحديث عند تفاعل المستخدم أو تغير البيانات

  • أمثلة

    Checkbox, TextField, Slider, وغيرها من الـ Widgetات التفاعلية

  • الأداء

    أثقل قليلاً لأنه يحتاج إلى إدارة حالة 

// مثال على Stateful Widget
class Counter extends StatefulWidget {
  @override
  _CounterState createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int count = 0; // متغير الحالة
  
  void increment() {
    setState(() { // تحديث الواجهة عند تغير الحالة
      count++;
    });
  }
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Count: $count'),
        ElevatedButton(
          onPressed: increment,
          child: Text('Increment'),
        ),
      ],
    );
  }
}

code شرح Stateful Widget بالتفصيل 

// 1. إنشاء Stateful Widget
class MyWidget extends StatefulWidget {
  // يمكنك تمرير props هنا عبر constructor
  const MyWidget({Key? key}) : super(key: key);

  // 2. دالة createState() التي تعيد State object
  @override
  _MyWidgetState createState() => _MyWidgetState();
}

// 3. State class التي تحتوي على المنطق والواجهة
class _MyWidgetState extends State<MyWidget> {
  // 4. متغيرات الحالة
  int _counter = 0;
  bool _isActive = false;

  // 5. دالة لتعديل الحالة
  void _incrementCounter() {
    setState(() { // 6. setState() تخبر Flutter بإعادة بناء الواجهة
      _counter++; // تعديل الحالة
    });
  }

  // 7. دالة build() التي تعيد الواجهة
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Counter: $_counter'),
        Switch(
          value: _isActive,
          onChanged: (value) {
            setState(() {
              _isActive = value;
            });
          },
        ),
        ElevatedButton(
          onPressed: _incrementCounter,
          child: Text('Increment'),
        ),
      ],
    );
  }
}

info شرح كل سطر:

  1. إنشاء Stateful Widget كصنف يرث من StatefulWidget
  2. دالة createState() التي تنشئ وتعيد State object
  3. State class التي تحتوي على بيانات الحالة ومنطق التعديل
  4. متغيرات الحالة التي يمكن تعديلها باستخدام setState()
  5. دوال لتعديل الحالة عند تفاعل المستخدم
  6. setState() تخبر Flutter بإعادة بناء الواجهة مع القيم الجديدة
  7. دالة build() التي تعيد واجهة المستخدم بناءً على الحالة الحالية

compare مقارنة بين Stateless و Stateful Widgets

المعيار Stateless Widget Stateful Widget
الحالة الداخلية لا يوجد يوجد (يمكن تغييرها)
إعادة البناء عند تغيير الـ parent فقط عند استدعاء setState()
الأمثلة Text, Icon, Container Checkbox, TextField, Slider
الأداء أخف وأسرع أثقل قليلاً

TextField و TextFormField

الـ TextField و TextFormField هما Widgets يستخدمان لـ إدخال النصوص من المستخدم مع إمكانية التحقق من صحة البيانات.

short_text TextField

  • حقل إدخال نصي أساسي

    يدعم التحكم في النص والتحقق البسيط

  • الخصائص الرئيسية

    controller, decoration, keyboardType, onChanged

  • حالات الاستخدام

    عند الحاجة إلى حقل إدخال بسيط بدون تحقق معقد

// مثال على TextField
TextEditingController _textController = TextEditingController();

TextField(
  controller: _textController,
  decoration: InputDecoration(
    labelText: 'اسم المستخدم',
    hintText: 'أدخل اسمك',
    border: OutlineInputBorder(),
    prefixIcon: Icon(Icons.person),
  ),
  keyboardType: TextInputType.text,
  onChanged: (value) {
    print('النص المدخل: $value');
  },
)

article TextFormField

  • حقل إدخال نصي متقدم

    يدعم التحقق من الصحة وحفظ البيانات

  • الخصائص الرئيسية

    validator, autovalidateMode, onSaved

  • حالات الاستخدام

    عند الحاجة إلى نماذج مع تحقق متقدم من الصحة

// مثال على TextFormField مع تحقق
final _formKey = GlobalKey<FormState>();
String _email = '';

TextFormField(
  decoration: InputDecoration(
    labelText: 'البريد الإلكتروني',
    hintText: 'example@domain.com',
    border: OutlineInputBorder(),
  ),
  keyboardType: TextInputType.emailAddress,
  validator: (value) {
    if (value == null || value.isEmpty) {
      return 'الرجاء إدخال البريد الإلكتروني';
    }
    if (!RegExp(r'^.+@.+\..+$').hasMatch(value)) {
      return 'البريد الإلكتروني غير صالح';
    }
    return null;
  },
  autovalidateMode: AutovalidateMode.onUserInteraction,
  onSaved: (value) {
    _email = value!;
  },
)

verified التحقق من صحة البيانات (Validation)

  • التحقق التلقائي (AutovalidateMode)

    يتحقق من الصحة أثناء الكتابة أو عند فقدان التركيز

  • دالة validator

    تحدد شروط الصحة وتعيد رسالة خطأ إذا لزم الأمر

  • دالة onSaved

    تخزن القيمة عند تقديم النموذج بنجاح

// مثال كامل على نموذج مع تحقق
class MyForm extends StatefulWidget {
  @override
  _MyFormState createState() => _MyFormState();
}

class _MyFormState extends State<MyForm> {
  final _formKey = GlobalKey<FormState>();
  final _passwordController = TextEditingController();

  @override
  Widget build(BuildContext context) {
    return Form(
      key: _formKey,
      child: Column(
        children: [
          TextFormField(
            decoration: InputDecoration(labelText: 'كلمة المرور'),
            obscureText: true,
            controller: _passwordController,
            validator: (value) {
              if (value == null || value.isEmpty) {
                return 'الرجاء إدخال كلمة المرور';
              }
              if (value.length < 6) {
                return 'يجب أن تكون كلمة المرور 6 أحرف على الأقل';
              }
              return null;
            },
          ),
          TextFormField(
            decoration: InputDecoration(labelText: 'تأكيد كلمة المرور'),
            obscureText: true,
            validator: (value) {
              if (value != _passwordController.text) {
                return 'كلمتا المرور غير متطابقتين';
              }
              return null;
            },
          ),
          ElevatedButton(
            onPressed: () {
              if (_formKey.currentState!.validate()) {
                _formKey.currentState!.save();
                ScaffoldMessenger.of(context).showSnackBar(
                  SnackBar(content: Text('تم التسجيل بنجاح')),
                );
              }
            },
            child: Text('تسجيل'),
          ),
        ],
      ),
    );
  }
}

التنقل بين الشاشات (Navigation)

في Flutter، يتم التعامل مع التنقل بين الشاشات باستخدام Navigator الذي يعتمد على مبدأ المكدس (Stack).

arrow_forward الانتقال إلى شاشة جديدة (push)

  • Navigator.push()

    يضيف شاشة جديدة إلى أعلى المكدس

  • الخصائص الرئيسية

    context, route (MaterialPageRoute)

  • حالات الاستخدام

    عند الانتقال إلى تفاصيل أو شاشة جديدة مع إمكانية العودة

// الانتقال إلى شاشة جديدة
Navigator.push(
  context,
  MaterialPageRoute(
    builder: (context) => DetailsScreen(),
  ),
);

// الانتقال باستخدام اسم الطريق (Named Route)
Navigator.pushNamed(
  context,
  '/details',
  arguments: {'id': 123}, // إرسال بيانات مع الطريق
);

arrow_back العودة إلى الشاشة السابقة (pop)

  • Navigator.pop()

    يزيل الشاشة الحالية من المكدس

  • إرجاع البيانات

    يمكن إرجاع بيانات إلى الشاشة السابقة

  • حالات الاستخدام

    عند الضغط على زر الرجوع أو إنهاء الشاشة

// العودة إلى الشاشة السابقة
Navigator.pop(context);

// العودة مع إرجاع بيانات
Navigator.pop(context, 'تم الحفظ بنجاح');

// استقبال البيانات في الشاشة السابقة
var result = await Navigator.push(
  context,
  MaterialPageRoute(builder: (context) => EditScreen()),
);
print('النتيجة: $result');

sync استبدال الشاشة الحالية (pushReplacement)

  • Navigator.pushReplacement()

    يستبدل الشاشة الحالية بشاشة جديدة

  • الاستخدام الشائع

    بعد تسجيل الدخول أو في تدفقات لا يمكن العودة منها

  • النسخة المسماة

    pushReplacementNamed() للطرق المسماة

// استبدال الشاشة الحالية
Navigator.pushReplacement(
  context,
  MaterialPageRoute(
    builder: (context) => HomeScreen(),
  ),
);

// النسخة المسماة
Navigator.pushReplacementNamed(context, '/home');

// مثال عملي بعد تسجيل الدخول
void _login() async {
  bool success = await authService.login(email, password);
  if (success) {
    Navigator.pushReplacementNamed(context, '/dashboard');
  }
}

alt_route إعداد الطرق المسماة (Named Routes)

  • المزايا

    تنظيم أفضل، إمكانية الانتقال من أي مكان، تمرير وسائط

  • الإعداد

    في MaterialApp أو WidgetApp باستخدام routes

  • الوسائط

    يمكن استقبالها عبر ModalRoute.of(context)!.settings.arguments

// إعداد الطرق المسماة في MaterialApp
MaterialApp(
  title: 'My App',
  initialRoute: '/',
  routes: {
    '/': (context) => HomeScreen(),
    '/login': (context) => LoginScreen(),
    '/profile': (context) => ProfileScreen(),
    '/details': (context) {
      // استقبال الوسائط
      final args = ModalRoute.of(context)!.settings.arguments as Map;
      return DetailsScreen(id: args['id']);
    },
  },
);

// الانتقال باستخدام الطرق المسماة
Navigator.pushNamed(
  context,
  '/details',
  arguments: {'id': 123},
);
