مفاهيم Dart المتقدمة

🔹 ما هو الـ Mixin؟

الـ mixin هو طريقة لإعادة استخدام الكود في أكثر من كلاس بدون الحاجة إلى الوراثة.

يشبه إلى حد كبير "الوراثة الأفقية": بدلاً من أن ترث كلاسًا واحدًا فقط، يمكنك استخدام أكثر من mixin في نفس الوقت، مما يمنحك مرونة كبيرة في التصميم.

🔸 لماذا نستخدم Mixin؟

  • Dart لا تدعم الوراثة المتعددة (multiple inheritance)، لكن باستخدام mixins يمكننا تجميع وظائف من أكثر من مصدر.
  • يجعل الكود أكثر نظافة وتنظيمًا.
  • يقلل التكرار.
  • يساعد في فصل المسؤوليات.

🔹 طريقة كتابة Mixin

✅ الصيغة الأساسية:

mixin CanFly {
  void fly() {
    print("I can fly!");
  }
}

✅ استخدامه في كلاس:

class Bird with CanFly {}

void main() {
  Bird b = Bird();
  b.fly(); // يطبع: I can fly!
}

🧠 ملاحظات مهمة عن الـ Mixin

النقطة الشرح
الكلمة المفتاحية mixin أو class عادي + with
لا يمكن إنشاء كائن مباشر من mixin لأنه ليس كائنًا مستقلاً
يمكن لكلاس استخدام أكثر من mixin مثال: class A with B, C
يمكن للـ mixin استخدام implements أو extends في حالات خاصة

🧪 مثال تطبيقي كامل

🎯 الهدف: لدينا حيوانات، بعضها يستطيع الطيران وبعضها لا.

// هذا هو الميكسين
mixin CanFly {
  void fly() {
    print("I can fly in the sky!");
  }
}

// كلاس أساسي
class Animal {
  String name;
  Animal(this.name);
}

// كائن يستخدم الميكسين
class Bird extends Animal with CanFly {
  Bird(String name) : super(name);
}

// كائن لا يستخدم الميكسين
class Cat extends Animal {
  Cat(String name) : super(name);
}

void main() {
  Bird b = Bird("Eagle");
  print(b.name); // Eagle
  b.fly(); // I can fly in the sky!

  Cat c = Cat("Kitty");
  print(c.name); // Kitty
  // c.fly(); ❌ لا يمكن، لأنها لا تستخدم CanFly
}

🧩 متى أستخدم Mixin وليس extends أو implements؟

تريد... استخدم
وراثة واحدة عادية extends
تطبيق واجهة فقط (بدون تنفيذ) implements
إعادة استخدام كود جاهز ✅ mixin

📌 نقاط إضافية

  • يمكن إضافة متغيرات ودوال داخل الـ mixin.
  • لا يمكن للـ mixin أن يكون فيه constructor.
  • يمكن استخدامه لأغراض متعددة مثل: تسجيل الأخطاء، التحقق من صلاحيات، قدرات معينة (مثل CanSwim, CanRun).

🎁 تدريب سريع

قم بإنشاء mixin يسمى Logger يحتوي على دالة log تطبع رسائل مع سجل زمني، ثم استخدمه في كلاس User.

mixin Logger {
  void log(String message) {
    print("Log: $message");
  }
}

class User with Logger {
  String username;
  User(this.username);
  
  void login() {
    log("$username has logged in.");
  }
}

void main() {
  User u = User("mostafa");
  u.login(); // Log: mostafa has logged in.
}

2. Abstract Class

الكلاس المجرد (Abstract Class) هو كلاس لا يمكن إنشاء كائنات منه مباشرة، ولكنه يستخدم كقاعدة للكلاسات الأخرى التي ترث منه.

abstract class Shape {
  void draw(); // Abstract method
  double getArea(); // Abstract method
}

class Circle extends Shape {
  double radius;
  Circle(this.radius);

  @override
  void draw() {
    print('Drawing Circle');
  }

  @override
  double getArea() {
    return 3.14 * radius * radius;
  }
}

الـ Enums

Enums (اختصار لـ Enumeration) هي طريقة لتعريف مجموعة ثابتة من القيم المحددة مسبقاً. تساعد في جعل الكود أكثر أماناً ووضوحاً.

// ❌ الطريقة التقليدية (بدون enum)
void printDayType(String day) {
  if (day == "Saturday" || day == "Sunday") {
    print("يوم عطلة");
  } else {
    print("يوم عمل");
  }
}

void main() {
  printDayType("Saturday");  // يوم عطلة ✅
  printDayType("Fridy");     // يوم عمل ❌ (خطأ إملائي)
}
// ✅ الطريقة الحديثة (باستخدام enum)
enum WeekDay {
  Saturday,
  Sunday,
  Monday,
  Tuesday,
  Wednesday,
  Thursday,
  Friday,
}

void printDayType(WeekDay day) {
  if (day == WeekDay.Saturday || day == WeekDay.Sunday) {
    print("يوم عطلة");
  } else {
    print("يوم عمل");
  }
}

void main() {
  printDayType(WeekDay.Saturday);  // يوم عطلة ✅
  printDayType(WeekDay.Friday);    // يوم عمل ✅
  
  // printDayType("Saturday");     // خطأ في الترجمة
  // printDayType(WeekDay.Fridy);  // خطأ في الترجمة
}

فوائد استخدام Enum

  • منع الأخطاء الإملائية: القيم محددة مسبقاً ولا يمكن كتابتها بشكل خاطئ
  • اقتراحات تلقائية: IDE سيقترح عليك القيم المتاحة عند الكتابة
  • أمان أعلى: لا يمكن تمرير قيم غير متوقعة
  • كود أكثر وضوحاً: القيم محددة وذات معنى واضح
  • سهولة الصيانة: التعديلات تتم في مكان واحد فقط

استخدامات متقدمة لـ Enum

// 1. الحصول على قائمة بالقيم
final allDays = WeekDay.values;

// 2. استخدام مع switch
String getArabicName(WeekDay day) {
  switch (day) {
    case WeekDay.Saturday:
      return "السبت";
    case WeekDay.Sunday:
      return "الأحد";
    // ... باقي الأيام
    default:
      return "يوم عمل";
  }
}

// 3. إضافة دوال إضافية باستخدام extension
extension WeekDayExtension on WeekDay {
  bool get isWeekend {
    return this == WeekDay.Saturday || this == WeekDay.Sunday;
  }
}

تمرين عملي

قم بتطوير المثال السابق بحيث:

  1. أضف خاصية `arabicName` ترجع اسم اليوم بالعربية
  2. أضف دالة `isWeekend` تحدد إذا كان اليوم عطلة أم لا
  3. استخدم `switch` في تنفيذ الدوال
// مثال للحل
void main() {
  print(WeekDay.Saturday.arabicName); // السبت
  print(WeekDay.Saturday.isWeekend);  // true
  print(WeekDay.Monday.isWeekend);    // false
}

شرح مفهوم الـ Interface في البرمجة

🎯 المثال العملي: جهاز يشحن أجهزة مختلفة

عندك شاحن (مثل كابل USB-C أو شاحن حائط) وعاوز تشحن أنواع أجهزة مختلفة:

  • 📱 موبايل
  • 💻 لابتوب
  • ⌚ ساعة ذكية

كل جهاز لازم يكون فيه طريقة charge() يشحن بها.

❌ المشكلة بدون استخدام Interface

لو عملت كلاس لكل جهاز بدون تنظيم، ممكن كل كلاس يعمل charge() بشكل مختلف، وفيه كلاس ممكن ينسى أصلاً يضيفها، أو يسميها charging() مثلاً.

ده هيسبب عدم التوحيد وصعوبة في التعامل مع كل الأجهزة.

🛠️ الحل: استخدام Interface

🔧 الخطوة 1: تعريف الـ Interface

abstract class Chargeable {
  void charge(); // أي جهاز بيشحن لازم ينفذ هذه الدالة
}

📱 كلاس الموبايل:

class Phone implements Chargeable {
  @override
  void charge() {
    print("📱 يتم شحن الموبايل الآن...");
  }
}

💻 كلاس اللابتوب:

class Laptop implements Chargeable {
  @override
  void charge() {
    print("💻 يتم شحن اللابتوب الآن...");
  }
}

⌚ كلاس الساعة الذكية:

class SmartWatch implements Chargeable {
  @override
  void charge() {
    print("⌚ يتم شحن الساعة الذكية الآن...");
  }
}

▶️ دالة main:

void main() {
  List<Chargeable> devices = [Phone(), Laptop(), SmartWatch()];

  for (var device in devices) {
    device.charge();
  }
}

✅ النتيجة:

📱 يتم شحن الموبايل الآن...
💻 يتم شحن اللابتوب الآن...
⌚ يتم شحن الساعة الذكية الآن...

✅ لماذا الـ Interface مفيد هنا؟

الفائدة التوضيح
كل الأجهزة تتبع نفس الشكل كلهم عندهم دالة charge()
سهولة التوسيع تقدر تضيف جهاز جديد (مثل سماعة) بسهولة
ضمان إن كل جهاز بيشحن فعلًا لغة البرمجة هتمنعك تنسى تنفذ charge()

5. Extension Functions

تسمح لنا بإضافة وظائف جديدة إلى الكلاسات الموجودة بدون تعديل الكلاس الأصلي.

extension NumberParsing on String {
  int toIntOrZero() {
    return int.tryParse(this) ?? 0;
  }
}

void main() {
  var numStr = '123';
  var num = numStr.toIntOrZero(); // 123
  print(num);
}

نصيحة عملية

استخدم Extension Functions عندما تريد إضافة وظائف مرتبطة بنوع معين ولكن لا تريد أو لا يمكنك تعديل الكلاس الأصلي.

مقدمة عن Flutter

6. ما هو Flutter؟

Flutter هو إطار عمل مفتوح المصدر من جوجل لإنشاء تطبيقات متعددة المنصات (Cross-Platform) بجودة عالية وسرعة كبيرة.

  • يعمل على نظامي Android و iOS بنفس الكود
  • يستخدم لغة Dart التي تعلمنا أساسياتها
  • يعتمد على مفهوم الويدجت (Widget) لبناء الواجهات
  • يقدم أداءً قريبًا من التطبيقات الأصلية (Native)
  • يدعم Hot Reload لتسريع عملية التطوير

7. مفهوم Widget

الويدجت (Widget) في Flutter هو الوحدة الأساسية لبناء واجهة المستخدم. كل شيء في Flutter هو ويدجت:

  • التخطيط (Layout) مثل Row و Column
  • العناصر (Elements) مثل Text و Image
  • أنماط التطبيق (Styling) مثل Padding و Margin
  • حتى التطبيق نفسه هو ويدجت

فكرة مهمة

في Flutter، الويدجتات تكون غير قابلة للتغيير (Immutable). عندما تحتاج لتغيير واجهة المستخدم، تقوم بإنشاء ويدجت جديد بدلاً من تعديل الويدجت الحالي.

8. Widget Tree

شجرة الويدجت (Widget Tree) هي الهيكل الهرمي للويدجتات التي تشكل واجهة المستخدم في تطبيق Flutter.

MaterialApp(
  home: Scaffold(
    appBar: AppBar(
      title: Text('My App'),
    ),
    body: Center(
      child: Column(
        children: [
          Text('Hello World'),
          Icon(Icons.favorite),
        ],
      ),
    ),
  ),
)

9. Stateless Widget

الويدجت عديم الحالة (Stateless Widget) هو ويدجت لا يتغير مظهره أثناء وقت التشغيل.

class MyText extends StatelessWidget {
  final String text;
  MyText(this.text);

  @override
  Widget build(BuildContext context) {
    return Text(text);
  }
}

متى نستخدم Stateless Widget؟

استخدم Stateless Widget عندما يكون مظهر الويدجت ثابتًا ولا يعتمد على بيانات متغيرة أو تفاعل المستخدم.

10. ويدجتات أساسية

الويدجت الوصف مثال
MaterialApp ويدجت أساسي لتطبيقات Material Design التطبيق الرئيسي
Scaffold يوضع هيكل أساسي للصفحة يحتوي على AppBar و Body
AppBar شريط العنوان في أعلى الصفحة يظهر عنوان التطبيق
Text عرض نص على الشاشة Text('Hello')
Container حاوية لتجميع وتنسيق الويدجتات يمكن إضافة Padding و Margin
Row ترتيب الويدجتات أفقياً صف من الأزرار
Column ترتيب الويدجتات رأسياً قائمة من العناصر
Icon عرض أيقونة Icon(Icons.home)
Image عرض صورة Image.network(url)

11. Hot Reload vs Hot Restart

  • Hot Reload:
    • يحتفظ بحالة التطبيق
    • سريع جداً (أقل من ثانية)
    • يستخدم عند تعديل واجهة المستخدم
    • لا يعيد تشغيل main()
  • Hot Restart:
    • يعيد تشغيل التطبيق من البداية
    • أبطأ من Hot Reload
    • يستخدم عند تعديل المنطق أو الحالة الأولية
    • يعيد تشغيل main()
