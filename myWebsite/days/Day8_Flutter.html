alt_route إعداد الطرق المسماة (Named Routes)

  • المزايا

    تنظيم أفضل، إمكانية الانتقال من أي مكان، تمرير وسائط

  • الإعداد

    في MaterialApp أو WidgetApp باستخدام routes

  • الوسائط

    يمكن استقبالها عبر ModalRoute.of(context)!.settings.arguments 

// إعداد الطرق المسماة في MaterialApp
MaterialApp(
  title: 'My App',
  initialRoute: '/',
  routes: {
    '/': (context) => HomeScreen(),
    '/login': (context) => LoginScreen(),
    '/profile': (context) => ProfileScreen(),
    '/details': (context) {
      // استقبال الوسائط
      final args = ModalRoute.of(context)!.settings.arguments as Map;
      return DetailsScreen(id: args['id']);
    },
  },
);

// الانتقال باستخدام الطرق المسماة
Navigator.pushNamed(
  context,
  '/details',
  arguments: {'id': 123},
);

ShowDialog

ShowDialog هي وظيفة في Flutter تستخدم لعرض نافذة حوار فوق الواجهة الحالية. تستخدم لعرض رسائل مهمة أو طلب إدخال من المستخدم.

info خصائص ShowDialog

  • context: السياق الحالي (مطلوب)
  • builder: دالة تبني محتوى الحوار
  • barrierDismissible: هل يمكن إغلاق الحوار بالنقر خارجها؟

lightbulb حالات الاستخدام

  • عرض رسائل تأكيد
  • طلب إدخال بيانات من المستخدم
  • إظهار تفاصيل إضافية
  • عرض أخطاء أو تحذيرات
// مثال على ShowDialog بسيط
showDialog(
  context: context,
  builder: (BuildContext context) {
    return AlertDialog(
      title: Text('تنبيه'),
      content: Text('هل أنت متأكد من حذف هذا العنصر؟'),
      actions: [
        TextButton(
          onPressed: () => Navigator.pop(context),
          child: Text('إلغاء'),
        ),
        TextButton(
          onPressed: () {
            // تنفيذ الحذف
            Navigator.pop(context);
          },
          child: Text('حذف'),

      ],
    );
  },
);

// حوار مخصص
showDialog(

          ElevatedButton(

  ),

SnackBar

SnackBar هي رسالة عابرة تظهر في أسفل الشاشة لتقديم ملاحظات للمستخدم حول عملية ما. تختفي تلقائيًا بعد فترة زمنية قصيرة.

info خصائص SnackBar

  • content: محتوى الرسالة (عادة Text Widget)
  • duration: مدة عرض الرسالة (مثال: Duration(seconds: 3))
  • action: زر إجراء إضافي (اختياري)
  • backgroundColor: لون خلفية الرسالة

warning ملاحظات مهمة

  • يجب أن يكون هناك Scaffold في شجرة الـ Widgetات لتعمل SnackBar
  • لإظهار SnackBar، نستخدم ScaffoldMessenger.of(context).showSnackBar()
  • تستخدم للرسائل غير الحرجة التي لا تتطلب تدخل المستخدم

code أمثلة على SnackBar 

// مثال بسيط
ScaffoldMessenger.of(context).showSnackBar(
  SnackBar(
    content: Text('تم الحفظ بنجاح!'),
    duration: Duration(seconds: 2),
  ),
);

// مع زر إجراء
ScaffoldMessenger.of(context).showSnackBar(
  SnackBar(
    content: Text('تم حذف العنصر'),
    duration: Duration(seconds: 4),
    action: SnackBarAction(
      label: 'تراجع',
      onPressed: () {
        // تنفيذ التراجع عن الحذف
      },
    ),
  ),
);

// مخصص
ScaffoldMessenger.of(context).showSnackBar(
  SnackBar(
    content: Row(
      children: [
        Icon(Icons.check, color: Colors.white),
        SizedBox(width: 8),
        Text('تمت العملية بنجاح'),
      ],
    ),
    backgroundColor: Colors.green,
    duration: Duration(seconds: 3),
  ),
);

ListView

ListView هو Widget يستخدم لعرض قائمة من العناصر التي يمكن التمرير فيها رأسيًا (أو أفقيًا). يأتي ListView بعدة أشكال لتناسب احتياجات التطبيق المختلفة.

format_list_bulleted أنواع ListView

  • ListView (العادي)

    عرض قائمة من العناصر المحددة مسبقًا

  • ListView.builder

    عرض قائمة ديناميكية بناءً على مصدر بيانات

  • ListView.separated

    مثل builder مع إمكانية إضافة فواصل مخصصة

  • ListView.custom

    للتحكم الكامل في كيفية عرض العناصر

compare مقارنة بين أنواع ListView

النوع الاستخدام الأداء
ListView عند وجود عدد صغير من العناصر الثابتة يحمّل كل العناصر مرة واحدة
ListView.builder للبيانات الديناميكية أو الكبيرة يحمّل العناصر عند الحاجة فقط (Lazy Loading)
ListView.separated عند الحاجة لفاصل مخصص بين العناصر مثل builder مع تحكم إضافي
ListView.custom للحالات المتقدمة التي تحتاج تحكم كامل يعتمد على التطبيق

GridView

GridView هو Widget يستخدم لعرض العناصر في تخطيط شبكي (Grid) مع إمكانية التمرير. مثالي لعرض مجموعات من العناصر المتشابهة مثل الصور أو المنتجات.

grid_view أنواع GridView

  • GridView (العادي)

  • GridView.count

    يحدد عدد الأعمدة مباشرة

  • GridView.extent

    يحدد أقصى عرض للعمود

مقارنة بين ListView و GridView

المعيار ListView GridView
التخطيط قائمة عمودية/أفقية شبكة بأعمدة وصفوف
الاستخدام الأمثل لعرض قوائم طويلة من البيانات لعرض مجموعات من العناصر المتشابهة
الأداء ممتاز للقوائم الطويلة (مع builder) جيد لكن يستهلك ذاكرة أكثر للعناصر المرئية
التخصيص سهل لعرض عناصر مختلفة الأشكال أفضل لعرض عناصر متساوية في الحجم
حالات الاستخدام قوائم المحادثات، الإعدادات، القوائم معارض الصور، المنتجات، الأيقونات

help متى تستخدم ListView ومتى تستخدم GridView؟

list استخدم ListView عندما:

  • تحتاج عرض قائمة طويلة من العناصر
  • العناصر لها ارتفاعات مختلفة
  • تحتاج تمريرًا سلسًا في اتجاه واحد
  • تعرض بيانات نصية أو عناصر معقدة
  • تحتاج إلى تفاعلات فردية مع كل عنصر

grid_on استخدم GridView عندما:

  • تعرض مجموعات من العناصر المتشابهة
  • كل العناصر لها نفس الحجم تقريبًا
  • تحتاج عرضًا مرئيًا منظمًا في شبكة
  • تعرض صورًا أو منتجات أو أيقونات
  • تحتاج استغلال المساحة بشكل أفضل

Future في Flutter

Future تمثل عملية غير متزامنة (Asynchronous) قد تكتمل الآن أو في المستقبل. تستخدم للعمليات التي تستغرق وقتًا مثل جلب البيانات من الإنترنت أو قراءة الملفات.

info المفاهيم الأساسية

  • Future.delayed

    تنفيذ كود بعد تأخير زمني

  • async/await

    لجعل الكود يبدو متزامنًا مع الحفاظ على عدم التزامن

  • then/catchError

    طريقة بديلة للتعامل مع Future

  • Future.any

    تنفيذ عدة Futures وأخذ أول نتيجة تكتمل

lightbulb لماذا نستخدم Future؟

  • لعدم تجميد واجهة المستخدم أثناء انتظار العمليات الطويلة
  • للتعامل مع العمليات غير المؤكدة النجاح
  • لتنفيذ مهام في الخلفية
  • للتعامل مع البيانات التي تحتاج وقتًا للتحميل

code أمثلة أساسية 

// مثال 1: Future.delayed
void main() {
  print("مرحبًا");
  Future.delayed(Duration(seconds: 2), () => print("هذا يظهر بعد ثانيتين"));
  print("مع السلامة");
}

// مثال 2: Future في دالة
Future myAsyncFunction() {
  return Future.delayed(Duration(seconds: 2), () {
    print('هذه رسالة بعد انتظار لمدة ثانيتين.');
  });
}

void main() {
  print("بداية البرنامج");
  myAsyncFunction();
  print("نهاية البرنامج");
}

warning مشكلة شائعة وحلها

عند محاولة تعيين Future إلى متغير مباشرة دون await، ستحصل على Future بدلاً من القيمة الفعلية. الحل هو استخدام async/await.

Future<String> loadFromCache() async {

void main() async {

ما هي API؟

API (واجهة برمجة التطبيقات) هي مجموعة من القواعد والبروتوكولات التي تسمح للتطبيقات بالتواصل مع بعضها البعض. في تطبيقات الهاتف، نستخدم APIs للتفاعل مع الخوادم البعيدة والحصول على البيانات أو تعديلها.

layers مهام API الرئيسية

  • جلب البيانات من الخادم (GET)
  • إرسال بيانات جديدة للخادم (POST)
  • تحديث بيانات موجودة على الخادم (PUT/PATCH)
  • حذف بيانات من الخادم (DELETE)
  • المصادقة والتحقق من الهوية

حزمة HTTP في Flutter

حزمة HTTP في Flutter تمكننا من إرسال طلبات الشبكة إلى الخوادم البعيدة. إليك كيفية إضافتها واستخدامها في مشروعك.

download طرق إضافة حزمة HTTP

الطريقة الأولى: باستخدام الأمر

flutter pub add http

هذا الأمر سيضيف الحزمة تلقائيًا إلى ملف pubspec.yaml

الطريقة الثانية: يدويًا

1. افتح ملف pubspec.yaml

2. أضف تحت dependencies:

dependencies:
  http: ^0.13.4

3. ثم نفذ الأمر في Terminal:

flutter pub get

code كيفية استخدام الحزمة 

// 1. استيراد الحزمة
import 'package:http/http.dart' as http;

// 2. إرسال طلب GET
Future fetchData() async {
  final response = await http.get(
    Uri.parse('https://api.example.com/data'),
  );
  
  if (response.statusCode == 200) {
    // معالجة البيانات
    print(response.body);
  } else {
    // معالجة الخطأ
    throw Exception('Failed to load data');
  }
}

// 3. إرسال طلب POST
Future sendData() async {
  final response = await http.post(
    Uri.parse('https://api.example.com/data'),
    headers: {'Content-Type': 'application/json'},
    body: jsonEncode({'key': 'value'}),
  );
  
  if (response.statusCode == 201) {
    print('Data sent successfully');
  }
}

warning ملاحظات مهمة

  • تأكد من وجود اتصال بالإنترنت قبل إرسال الطلبات
  • استخدم try-catch للتعامل مع الأخطاء المحتملة
  • تحقق دائمًا من statusCode قبل معالجة الاستجابة
  • استخدم async/await للتعامل مع العمليات غير المتزامنة

2. دالة جلب المشاركات (GET)

Future fetchPosts() async {
  setState(() => isLoading = true);
  final response = await http.get(
    Uri.parse('https://jsonplaceholder.typicode.com/posts')
  );

  if (response.statusCode == 200) {
    setState(() {
      posts = json.decode(response.body);
      isLoading = false;
    });
    debugPrint('تم تحميل ${posts.length} مشاركات');
  } else {
    setState(() => isLoading = false);
    showError('فشل في جلب البيانات');
  }
}

3. دالة إنشاء مشاركة جديدة (POST)

Future createPost(String title, String body) async {
  final response = await http.post(
    Uri.parse('https://jsonplaceholder.typicode.com/posts'),
    headers: {'Content-Type': 'application/json'},
    body: jsonEncode({
      'title': title,
      'body': body,
      'userId': 1,
    }),
  );

  if (response.statusCode == 201) {
    setState(() {
      posts.insert(0, json.decode(response.body));
    });
    debugPrint('تمت إضافة مشاركة جديدة');
  } else {
    showError('فشل في إضافة المشاركة');
  }
}

4. دالة تعديل مشاركة (PUT)

Future updatePost(int id, String newTitle, String newBody) async {
  final response = await http.put(
    Uri.parse('https://jsonplaceholder.typicode.com/posts/$id'),
    headers: {'Content-Type': 'application/json'},
    body: jsonEncode({
      'id': id,
      'title': newTitle,
      'body': newBody,
      'userId': 1,
    }),
  );

  if (response.statusCode == 200) {
    setState(() {
      int index = posts.indexWhere((post) => post['id'] == id);
      if (index != -1) posts[index] = json.decode(response.body);
    });
  } else {
    showError('فشل في تعديل المشاركة');
  }
}

5. دالة حذف مشاركة (DELETE)

Future deletePost(int id) async {
  // طلب تأكيد من المستخدم قبل الحذف
  bool confirm = await showDialog(
    context: context,
    builder: (context) => AlertDialog(
      title: Text('تأكيد الحذف'),
      content: Text('هل أنت متأكد من حذف هذه المشاركة؟'),
      actions: [
        TextButton(
          onPressed: () => Navigator.pop(context, false),
          child: Text('إلغاء'),
        ),
        ElevatedButton(
          onPressed: () => Navigator.pop(context, true),
          child: Text('حذف'),
        ),
      ],
    ),
  );

  if (confirm != true) return;

  final response = await http.delete(
    Uri.parse('https://jsonplaceholder.typicode.com/posts/$id')
  );

  if (response.statusCode == 200) {
    setState(() => posts.removeWhere((post) => post['id'] == id));
    ScaffoldMessenger.of(context).showSnackBar(
      SnackBar(content: Text('تم الحذف بنجاح')),
    );
  } else {
    showError('فشل في حذف المشاركة');
  }
}

void showAddPostDialog() {
  final titleController = TextEditingController();
  final bodyController = TextEditingController();

  showDialog(
    context: context,
    builder: (context) => AlertDialog(
      title: Text('إضافة مشاركة جديدة'),
      content: Column(
        mainAxisSize: MainAxisSize.min,
        children: [
          TextField(
            controller: titleController,
            decoration: InputDecoration(labelText: 'العنوان'),
          ),
          TextField(
            controller: bodyController,
            decoration: InputDecoration(labelText: 'النص'),
          ),
        ],
      ),
      actions: [
        TextButton(
          onPressed: () => Navigator.pop(context),
          child: Text('إلغاء'),
        ),
        ElevatedButton(
          onPressed: () {
            if (titleController.text.isNotEmpty && 
                bodyController.text.isNotEmpty) {
              createPost(titleController.text, bodyController.text);
              Navigator.pop(context);
            } else {
              showError('يرجى ملء جميع الحقول');
            }
          },
          child: Text('إضافة'),
        ),
      ],
    ),
  );
}

ملخص النقاط الرئيسية

api APIs

  • واجهة للتواصل بين التطبيق والخادم
  • أنواع الطلبات: GET, POST, PUT, DELETE
  • تستخدم JSON كتنسيق رئيسي للبيانات
  • تحتاج إلى معالجة الأخطاء دائمًا

http حزمة HTTP

  • إضافتها عبر: flutter pub add http
  • استيرادها: import 'package:http/http.dart'
  • الدوال الرئيسية: get(), post(), put(), delete()
  • تتعامل مع العمليات غير المتزامنة

code أفضل الممارسات

  • تحقق دائمًا من statusCode
  • استخدم async/await للعمليات غير المتزامنة
  • أضف تحميل أثناء انتظار الاستجابة
  • أضف معالجة للأخطاء باستخدام try-catch
  • استخدم Model classes لتنظيم البيانات
